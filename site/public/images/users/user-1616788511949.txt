 ------ PASOS PARA MANEJARSE ENTRE RAMAS CON GIT ------

1) Al comenzar un proyecto se debe crear la rama develop

	git checkout -b develop

Las ramas nuevas contendran los contenidos de la rama de la que nacieron.
Develop contendra el trabajo funcional aportado y ya desteado de cada integrante

2) Para moverse de una rama a otra

	git checkout nombre-de-la-rama

3) A partir de la rama develop crear rama propia

	git checkout -b tarea/mi-nombre

Hacer los pull y push a esta rama durante el trabajo designado
AL PUSHEAR SIEMPRE ESPECIFICAR EL NOMBRE DE LA RAMA EN LA QUE ESTOY TRABAJANDO
EJ:	git push origin tarea/mi-nombre

 ----- PROCEDIMIENTO PARA FUCIONAR ----

1) Antes que nada pullear los cambios de la rama develop repo remoto al local con:

(estando en develop)	git pull origin develop

Esto se realiza para estar seguros de que nos traemos el estado actual de la rama.
Por si algun colaborador ya subio cambios a develop y no pisarlos al pushear
nuestros cambios desactualizados.

2) Al tener el trabajo terminado y la develop actualizada, se debe fucionar primero
mi rama con la develop. Para esto debo estar en mi rama y ejecutar: 

	git merge develop

Esto se hace para prevenir que halla conflictos en el codigo.
Si los hay es preferible que ocurran en mi rama de trabajo y no en la develop.

En el caso de que llegara a haber conflicto y no saber como solucionarlo y se 
tenga que recorrir a la consulta de algun compañero, se puede cancelar el merge con:

	git merge --abort 

3) Una vez tengamos EN NUESTRA RAMA el trabajo terminado, fucionado con develop y
totalmente funcional (sin errores o conflictos), fucionar develop con nuestra rama.
Para esto hay que ESTAR EN LA RAMA DEVELOP y mergear con:
	
	git merge tarea/mi-nombre

4) Finalmente pushear develop al repo

	git push origin develop

NOTA: Es recomendable tambien, por seguridad, subir nuestra rama actualizada y no
borrarla hasta mediados del proximo sprint.

----- FUCIONAR CON RAMA MAIN O MASTER ----

1) Al finalizar las tareas del sprint, el proyecto terminado debería quedar en la
la rama develop por lo que, solo queda fucionar la rama main con develop.
ESTANDO EN LA RAMA MAIN ejecutar:
	
	git merge develop

	        Y
	
	git push origin main
